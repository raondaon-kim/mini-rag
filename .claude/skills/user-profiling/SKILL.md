# 사용자 프로파일링 가이드

## 자동 감지 트리거

대화에서 다음 패턴을 감지하면 **즉시** mcp__memory에 저장하세요.

### 1. 사용자 정보 (Entity: User)
| 감지 패턴 | 저장할 정보 | 예시 |
|----------|-----------|------|
| "저는 ~입니다", "제 이름은" | 이름, 역할 | "저는 김팀장입니다" |
| "~부서", "~팀" | 소속 부서 | "마케팅팀에서 일해요" |
| "~업무를 하고 있어" | 주요 업무 | "교육과정 설계를 하고 있어" |
| "~회사", "우리 회사는" | 회사 정보 | "AIED 회사야" |
| "~을 좋아해", "~선호" | 선호도 | "표로 정리해주는 게 좋아" |

### 2. 회사/조직 정보 (Entity: Company)
| 감지 패턴 | 저장할 정보 |
|----------|-----------|
| "우리 회사는 ~", "우리는 ~회사야" | 회사 유형, 산업 |
| "~사업을 하고 있어" | 사업 영역 |
| "~서비스", "~제품" | 주요 제품/서비스 |
| "~명", "~팀이 있어" | 조직 규모 |

### 3. 업무 패턴 (Entity: WorkPattern)
| 감지 패턴 | 저장할 정보 |
|----------|-----------|
| 반복 요청 (3회+) | 자주 하는 업무 유형 |
| 특정 포맷 선호 | 선호 문서 포맷 (PPT/엑셀/워드) |
| 특정 프레임워크 요청 | 선호 분석 프레임워크 |
| 피드백 ("이건 좋았어", "다음엔 ~") | 품질 기준 |

### 4. 프로젝트 정보 (Entity: Project)
| 감지 패턴 | 저장할 정보 |
|----------|-----------|
| "~프로젝트 하고 있어" | 프로젝트명, 목적 |
| "~까지 마감이야" | 일정, 마감일 |
| "~과 협업 중" | 협업자/이해관계자 |

## 저장 방법

### mcp__memory 엔티티 구조
```
Entity: User
  - name: 사용자 이름
  - Observations: 역할, 부서, 선호도, 업무 스타일

Entity: Company  
  - name: 회사명
  - Observations: 산업, 사업 영역, 규모, 특징

Entity: WorkPattern
  - name: "사용자 업무 패턴"
  - Observations: 자주 요청하는 업무, 선호 포맷, 피드백 내용

Relation: User → works_at → Company
Relation: User → prefers → WorkPattern
```

### 저장 시점
- **즉시**: 이름, 회사, 역할 등 명시적 정보
- **3회 반복 시**: 업무 패턴 (같은 유형 요청 3회 이상)
- **피드백 시**: 품질 기준 ("더 짧게", "표로", "한국어로")

## 활용 방법 (대화 시작 시)

오케스트레이터는 매 대화 시작 시:
1. `mcp__memory__search_nodes`로 "User", "Company", "WorkPattern" 검색
2. 결과를 대화 컨텍스트에 포함
3. 이전에 저장된 선호도/패턴에 맞춰 응답 조정

### 개인화 적용 예시
| 저장된 정보 | 적용 방법 |
|-----------|---------|
| "AIED 회사" | 교육 관련 요청 → education-specialist 우선 |
| "표로 정리 선호" | 분석 결과를 마크다운 표로 기본 제공 |
| "한국어 + 영어 병기" | 전문 용어에 영어 원문 병기 |
| "자주 PPT 요청" | 분석 후 "PPT로 만들어드릴까요?" 제안 |
| "마감 다음주 금요일" | 긴급도 반영, 핵심만 빠르게 |

## 프라이버시 규칙
- 민감 정보 (급여, 개인 연락처) 저장 금지
- 사용자가 "잊어줘", "삭제해줘" 요청 시 즉시 삭제
- 저장된 정보는 해당 사용자 세션에서만 활용
