# 비즈니스 이메일 작성

## 이메일 유형별 구조

### 요청 이메일
```
제목: [행동 동사] + [구체적 내용] (예: "Q3 예산안 검토 요청")
BLUF: [핵심 요청 1문장]
배경: [왜 필요한지 2-3문장]
상세: [구체적 요청 사항]
기한: [언제까지 필요한지]
감사: [1문장]
```

### 보고 이메일
```
제목: [보고] + [주제] + [기간]
BLUF: [핵심 결과 1문장]
주요 성과: [3-5개 불릿]
이슈: [있으면]
다음 단계: [행동 계획]
```

### 불만/클레임 대응
```
공감: "불편을 드려 죄송합니다"
사실 확인: [조사 결과]
해결 방안: [구체적 조치]
재발 방지: [개선 약속]
```

### 콜드 아웃리치
```
훅: [상대방의 최근 성과/이슈 언급]
가치: [내가 줄 수 있는 것 1문장]
증거: [유사 사례/숫자]
CTA: [구체적, 부담 없는 다음 단계]
- "15분 통화" > "미팅 잡기"
```

## 톤 가이드
| 상황 | 톤 | 예시 |
|------|---|------|
| 상사에게 | 공손+직접적 | "검토 부탁드립니다" |
| 동료에게 | 친근+명확 | "확인해주실 수 있을까요?" |
| 고객에게 | 전문적+따뜻 | "도움이 필요하시면 말씀해주세요" |
| 외부에게 | 격식+간결 | "귀사의 검토를 요청드립니다" |

## 체크리스트
- [ ] 제목이 내용을 즉시 알려주는가?
- [ ] 첫 문장이 핵심 메시지인가? (BLUF)
- [ ] 요청 사항이 명확한가? (누가, 무엇을, 언제)
- [ ] 5문장 이내로 읽히는가? (긴 이메일은 문서로)
- [ ] 톤이 상황에 맞는가?

## 프로액티브 트리거
- 제목이 불명확하여 내용을 알 수 없음
- 본문이 5줄을 초과함 (BLUF 원칙 위반)
- 수신자/CC가 불분명함
- 요청 사항이 본문 마지막에 묻혀 있음
- 톤이 상황에 부적절함
