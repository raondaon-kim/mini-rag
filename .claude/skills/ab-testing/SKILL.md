# A/B 테스트 가이드

## 테스트 설계 프레임워크

### 1. 가설 수립 (PIE Framework)
```
만약 [변경 사항]을 적용하면,
[측정 지표]가 [방향]할 것이다.
왜냐하면 [근거/데이터].
```

**우선순위 기준**:
- **P**otential: 개선 가능성 (1-10)
- **I**mportance: 비즈니스 중요도 (1-10)
- **E**ase: 구현 용이성 (1-10)

### 2. 테스트 설정
| 항목 | 내용 |
|------|------|
| 테스트 대상 | 헤드라인/CTA/레이아웃/가격/이미지 |
| 대조군 (A) | 현재 버전 |
| 실험군 (B) | 변경 버전 |
| 1차 지표 | 전환율, CTR, 매출 등 (1개만) |
| 2차 지표 | 보조 측정 지표 |
| 표본 크기 | 통계 계산기로 산출 |
| 기간 | 최소 1-2주 (요일 사이클 포함) |
| 신뢰 수준 | 95% (p < 0.05) |

### 3. 표본 크기 계산
```
필요 표본 = f(기저 전환율, 최소 감지 효과, 검정력, 신뢰수준)

예시: 기저 전환율 3%, 최소 10% 상대 개선 감지
→ 각 그룹당 약 30,000명 필요
```

### 4. 결과 분석 템플릿
| 지표 | 대조군 (A) | 실험군 (B) | 차이 | p-value | 판정 |
|------|-----------|-----------|------|---------|------|
| 전환율 | | | | | |
| 수익/사용자 | | | | | |
| 이탈률 | | | | | |

### 5. 의사결정
- **p < 0.05 + 실질적 효과**: 실험군 적용
- **p < 0.05 + 미미한 효과**: 비용 대비 판단
- **p >= 0.05**: 추가 데이터 수집 또는 새 가설

## 주의사항
- 한 번에 하나만 변경 (다변량 테스트는 별도)
- 테스트 중간에 결과 보고 성급한 판단 금지
- 외부 변수(시즌, 이벤트) 통제

## 프로액티브 트리거
- 가설 없이 테스트를 시작할 때 → PIE 프레임워크로 가설 수립 먼저 진행
- 표본 크기를 산정하지 않았을 때 → 통계 계산기로 필요 표본 크기 산출
- 1차 지표가 2개 이상일 때 → 1차 지표 1개로 축소, 나머지는 2차 지표로 분류
- 테스트 기간이 1주 미만일 때 → 최소 1-2주(요일 사이클 포함)로 연장
- 중간에 결과를 보고 성급한 판단을 할 때 → 사전 설정한 기간/표본 충족까지 대기
