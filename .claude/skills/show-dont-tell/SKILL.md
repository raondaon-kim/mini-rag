# Show, Don't Tell 가이드

## 핵심 원칙
"말하지 말고 보여줘라" — 감정이나 상태를 직접 서술하지 말고, 독자가 스스로 느끼게 하라.

## 변환 규칙

### 감정 → 행동/감각
| Tell (❌) | Show (✓) |
|----------|---------|
| 그녀는 화가 났다 | 그녀는 책상을 내리쳤다. 손이 벌겋게 달아올랐다. |
| 회의실 분위기가 무거웠다 | 아무도 먼저 말을 꺼내지 않았다. 에어컨 소리만 웅웅거렸다. |
| 성과가 좋았다 | 분기 매출이 전년 대비 47% 증가했다. |

### 적용 우선순위
1. **도입부**: 첫 문장에서 장면/상황을 보여줌
2. **사례/케이스**: 추상 → 구체적 숫자/에피소드
3. **결론**: 임팩트를 수치/비유로 체감

## 비즈니스 글쓰기 적용
| Tell (❌) | Show (✓) |
|----------|---------|
| 시장이 크다 | 2025년 글로벌 시장 규모는 $420B이다 |
| 고객이 만족했다 | NPS 78점, 재구매율 92% |
| 기술력이 뛰어나다 | 응답 속도 50ms, 가동률 99.99% |

## 체크리스트
- [ ] "~했다", "~이다" 직접 서술 → 장면/데이터로 교체
- [ ] 형용사/부사 최소화 → 동사/명사로 구체화
- [ ] 추상적 주장 → 숫자/사례/비유

## 프로액티브 트리거
- 도입부가 추상적 서술로 시작 → 구체적 장면/데이터로 교체
- "좋은", "뛰어난", "성공적인" 등 형용사 남발 → 수치/사례로 대체
- 감정을 직접 명명 ("화가 났다") → 행동/감각 묘사로 전환
- 비즈니스 문서에서 "시장이 크다" 식 주장 → 구체적 시장 규모 수치
- 결론에 임팩트가 없음 → 수치/비유로 체감 가능하게

## 산출물
- 개선된 문장/단락 (Before → After)
- Show, Don't Tell 적용 보고서
