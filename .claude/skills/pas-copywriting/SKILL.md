# PAS — Problem, Agitate, Solution

**출처**: Dan Kennedy, 다이렉트 리스폰스 카피라이팅 전통. 짧은 카피의 기본.

**적용 대상**: 이메일 제목+본문, SNS 포스트, 짧은 광고, 랜딩페이지 헤드라인

## 구조

### P — Problem (문제 제기)
```
독자의 고통점을 독자의 언어로 정확히 짚기
- 업계 용어 X, 독자가 실제로 쓰는 말 O
- "매번 야근하면서 보고서 쓰느라 지치지 않으셨나요?"
```

### A — Agitate (고통 증폭)
```
문제를 방치하면 어떻게 되는지 보여주기
- 감정적 + 논리적 양면
- "이대로 두면 다음 분기에도, 그 다음에도..."
- 공포 마케팅 금지 — 사실 기반 증폭
```

### S — Solution (해결책 제시)
```
고통에서 해방되는 느낌으로 전환
- 솔루션이 Problem과 직접 대응
- CTA 즉시 제시
```

## 예시 (이메일)

```
제목: 매주 12시간 보고서에 쓰고 계신가요?

P: 엑셀 취합, 서식 맞추기, 검토... 매주 같은 작업의 반복.
A: 그 시간이면 한 달에 48시간. 1년이면 약 600시간. 
   그 시간에 고객 미팅을, 전략 수립을, 휴식을 할 수 있었을 텐데.
S: [제품명]은 보고서를 3분 만에 자동 생성합니다. 
   → [무료 체험 시작하기]
```

## 품질 체크
- [ ] Problem이 독자의 언어로 쓰여졌는가?
- [ ] Agitate가 방치 비용을 구체적으로 보여주는가?
- [ ] Solution이 Problem과 직접 대응하는가?
- [ ] 전체가 30초 이내에 읽히는가?

## 프로액티브 트리거
- Problem이 추상적임 (구체적 상황/숫자가 필요)
- Agitate가 공포 마케팅 수준임 (적정선으로 조정 필요)
- Solution이 기능만 나열함 (가치/혜택 중심으로 전환 필요)
- CTA가 없거나 본문과 분리되어 있음
- Problem→Agitate→Solution 흐름이 끊김
