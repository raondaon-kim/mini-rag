# 컴플라이언스 감사 가이드

## 감사 프로세스

### 4단계 감사 사이클
```
1. 계획 → 2. 현장 감사 → 3. 보고 → 4. 후속 조치
```

### 1. 감사 계획
| 항목 | 내용 |
|------|------|
| 감사 목적 | 규제 준수 / 프로세스 효율 / 리스크 식별 |
| 감사 범위 | 부서, 프로세스, 기간 |
| 감사 기준 | ISO, 법규, 내부 정책 |
| 감사팀 | 리더 + 멤버 |
| 일정 | 시작~종료, 보고 기한 |

### 2. 감사 체크리스트 (범용)
| # | 점검 항목 | 준수 | 미준수 | N/A | 비고 |
|---|---------|------|--------|-----|------|
| 1 | 정책/절차 문서화 | | | | |
| 2 | 문서 최신성 유지 | | | | |
| 3 | 교육 기록 보관 | | | | |
| 4 | 접근 권한 관리 | | | | |
| 5 | 로그/기록 보존 | | | | |
| 6 | 개인정보 처리 동의 | | | | |
| 7 | 사고 보고 절차 | | | | |
| 8 | 변경 관리 절차 | | | | |

### 3. 감사 보고서 구조
1. **요약**: 전체 준수율, 핵심 발견사항
2. **범위/방법**: 감사 대상, 기준, 방법론
3. **발견사항**: 부적합(Major/Minor), 관찰사항, 모범사례
4. **권고사항**: 우선순위별 개선 과제
5. **부록**: 증빙, 인터뷰 기록

### 발견사항 분류
| 등급 | 정의 | 대응 기한 |
|------|------|----------|
| Major | 시스템적 미준수, 즉각 리스크 | 30일 |
| Minor | 단일 사례 미준수, 낮은 리스크 | 90일 |
| Observation | 개선 권고 | 다음 감사 |
| Best Practice | 모범 사례 공유 | - |

## 주요 프레임워크별 핵심 영역
| 프레임워크 | 핵심 점검 |
|-----------|---------|
| GDPR | 동의, 정보주체 권리, DPO, DPIA |
| SOC2 | 보안, 가용성, 기밀성, 처리무결성 |
| ISMS (ISO 27001) | 위험 평가, 통제, 사고 관리 |
| ISO 9001 | PDCA, 프로세스, 고객 만족 |

## 프로액티브 트리거
- 감사 범위가 정의되지 않았을 때 → 대상 부서, 프로세스, 기간 명확히 한정
- 점검 기준(법규/표준)이 명시되지 않았을 때 → 적용 법규/ISO 표준/내부 정책 목록 확정
- 발견사항에 등급 분류(Major/Minor/Observation)가 없을 때 → 리스크 기반 등급 부여
- 권고사항에 시정 기한이 없을 때 → 등급별 대응 기한(30일/90일/다음 감사) 설정
- 이전 감사 후속 조치 확인이 안 되었을 때 → 전회 감사 CAPA 이행 상태 점검

## 산출물
- 감사 보고서 (요약, 발견사항, 권고사항)
- 시정 조치 추적표 (CAPA 번호, 기한, 상태)
- 감사 체크리스트
