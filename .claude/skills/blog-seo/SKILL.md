# Blog & SEO Content — Inverted Pyramid + F-Pattern

**출처**: Jakob Nielsen, Nielsen Norman Group (F-Pattern, 2006) + Brian Dean, Backlinko (Skyscraper Technique, 2013) + 저널리즘 역피라미드

## 블로그 포스트 구조

```
1. 헤드라인 — 핵심 혜택/호기심 유발 (60자 이내)
2. 리드 문단 — 핵심 답변 즉시 제공 (역피라미드)
3. 목차/요약 — 긴 포스트는 클릭 가능한 목차
4. 본문 섹션들 — H2/H3로 구조화
   각 섹션: 소제목 → 핵심 → 상세 → 예시
5. 요약/결론 — 핵심 takeaway 3-5개 불릿
6. CTA — "다음 읽을 글", "뉴스레터 구독", "공유"
```

## F-Pattern 최적화 (독자는 F자로 스캔한다)

```
[████████████████████] ← 첫 2줄은 끝까지 읽음
[████████████        ] ← 두 번째 줄은 절반만
[████                ] ← 이후는 왼쪽만 스캔
[████                ]
[████                ]
```

**적용**:
- 첫 2문단에 가장 중요한 정보
- 소제목의 첫 2단어에 핵심어 배치
- 불릿 포인트는 핵심 단어로 시작
- 200-300 단어마다 소제목으로 리셋

## SEO 체크리스트

- [ ] 타겟 키워드가 제목, 첫 문단, H2에 자연스럽게 포함
- [ ] 메타 설명 150-160자 (핵심 혜택 + CTA)
- [ ] H2/H3 소제목이 검색 의도를 반영
- [ ] 내부 링크 2-3개, 외부 링크 1-2개
- [ ] 이미지에 alt 텍스트
- [ ] 문단 2-4문장 (웹 가독성)
- [ ] 전체 1,500단어 이상 (심층 콘텐츠)

## 품질 체크

- [ ] 첫 문단에서 독자의 핵심 질문에 답하는가?
- [ ] 언제든 읽기를 멈춰도 핵심 정보를 얻었는가?
- [ ] 소제목만 읽어도 전체 내용을 파악 가능한가?
- [ ] 경쟁 콘텐츠보다 더 깊거나 새로운 관점이 있는가?

## 프로액티브 트리거
- 타깃 키워드가 미정의 상태로 작성 시작
- 메타 디스크립션이 없거나 150자 미만/초과
- H2/H3 구조가 부족하여 스캔 가독성 저하
- 내부링크 계획이 없음 (사일로 구조 미반영)
- 독자 페르소나가 불명확하여 톤/깊이 불일치
- 첫 문단에서 핵심 답변을 제공하지 않음

## 산출물
- SEO 최적화 블로그 포스트 (H2/H3 구조 + 메타)
- 키워드 전략표 (메인 키워드 + 롱테일 후보)
- 내부링크 맵 (관련 콘텐츠 연결 계획)
- 품질 체크 + SEO 체크리스트 완료 리포트
