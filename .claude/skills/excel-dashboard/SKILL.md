# Excel Dashboard — Cole Nussbaumer + IBCS

**출처**: Cole Nussbaumer Knaflic "Storytelling with Data" + IBCS (International Business Communication Standards, SAP/Siemens 표준)

## 시트 구조

```
시트 1: "대시보드" (요약)
  - 핵심 KPI 3-5개 (큰 숫자 + 전기 대비 변동)
  - 추이 차트 1-2개
  - 상태 신호등 (🟢🟡🔴)

시트 2-N: "상세_[주제]" (데이터)
  - 각 KPI의 상세 데이터 테이블
  - 시트별 하나의 주제

시트 마지막: "Raw_Data" (원시 데이터, 선택)
```

## KPI 카드 형식

```
[KPI 이름]
[현재 값]         [변동]
$4.2M            ▲ 12% vs 전분기
```

## 핵심 원칙 (Cole Nussbaumer)

1. **맥락 이해**: 누가, 무엇을, 어떤 행동을?
2. **적절한 시각화**: 비교→막대, 추이→선, 구성→파이(주의)
3. **차트 정크 제거**: 3D 효과, 불필요한 그리드, 과도한 색상 X
4. **시선 유도**: 핵심 데이터에만 강조색 사용
5. **데이터 스토리**: 수치가 말하는 이야기를 명시적으로 서술

## IBCS 표기 규칙

| 항목 | 규칙 |
|------|------|
| 실적(Actual) | 진한 색상 |
| 계획(Plan) | 테두리선만 (빈 막대) |
| 전년(Previous Year) | 회색 |
| 증감 | ▲ 녹색, ▼ 빨간색 |
| 단위 | 항상 명시 (백만원, %, 명) |

## 프로액티브 트리거
- KPI가 5개를 초과할 때 → 3-5개로 축소, 나머지는 상세 시트로 이동
- 차트 유형이 데이터에 부적합할 때 → 비교=막대, 추이=선, 구성=파이 원칙 적용
- 필터/슬라이서가 없을 때 → 기간/부서/제품 등 주요 차원 필터 추가
- 색상이 의미 없이 사용될 때 → IBCS 표기 규칙에 따라 색상 체계 통일
- 데이터 소스 연결이 표시되지 않을 때 → Raw_Data 시트 출처 및 갱신일 명시
