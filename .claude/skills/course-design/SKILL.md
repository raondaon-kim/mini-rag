# 교육과정 설계 가이드

## 설계 모델 선택

| 모델 | 적합 상황 | 기간 |
|------|----------|------|
| **ADDIE** | 체계적 개발, 대규모, 인증 과정 | 8-16주 |
| **SAM** (Successive Approximation) | 빠른 반복, 프로토타입 중심 | 4-8주 |
| **Backward Design** | 성과 중심, 평가 먼저 설계 | 4-12주 |

## ADDIE 프레임워크

### 1. Analysis (분석)
| 항목 | 질문 |
|------|------|
| 학습자 | 사전 지식 수준, 학습 동기, 선호 방식 |
| 요구 | 현재 역량 vs 목표 역량 (Gap) |
| 환경 | 온라인/오프라인, 시간 제약, 도구 |
| 제약 | 예산, 기간, 인력 |

### 2. Design (설계)
**학습목표 작성 (ABCD 공식)**
```
A(Audience): [학습자]가
B(Behavior): [관찰 가능한 행동]을 할 수 있다
C(Condition): [주어진 조건] 하에서
D(Degree): [기준 수준]으로
```

**Bloom's Taxonomy 기반 동사**
| 수준 | 동사 | 예시 |
|------|------|------|
| 기억 | 나열, 정의, 식별 | "3가지 원칙을 나열할 수 있다" |
| 이해 | 설명, 요약, 비교 | "차이점을 설명할 수 있다" |
| 적용 | 활용, 실행, 계산 | "공식을 활용하여 계산할 수 있다" |
| 분석 | 구분, 분류, 추론 | "원인을 분석할 수 있다" |
| 평가 | 판단, 비평, 선택 | "적절한 전략을 판단할 수 있다" |
| 창조 | 설계, 구성, 개발 | "새로운 모델을 설계할 수 있다" |

### 3. Development (개발)
- 콘텐츠 제작 (교안, 슬라이드, 영상, 활동지)
- 프로토타입 → 파일럿 → 수정

### 4. Implementation (실행)
- 교수자 준비 (FTF/가이드)
- 학습자 오리엔테이션
- 학습관리시스템(LMS) 세팅

### 5. Evaluation (평가)
- **형성평가**: 과정 중 퀴즈, 피드백
- **총괄평가**: 최종 시험, 프로젝트
- **Kirkpatrick 4단계**: 반응 → 학습 → 행동 → 결과

## 프로액티브 트리거
- 학습목표에 관찰 가능한 동사가 없음 → "~이해한다" ✗ → "~설명할 수 있다" ✓
- 평가와 학습목표가 정렬되지 않음 → 목표별 평가 방법 확인
- 학습자 분석 없이 설계 시작 → 반드시 Analysis 먼저
- 차시당 학습목표 3개 초과 → 분리 권고
- 강의만 있고 활동이 없음 → 능동적 학습 활동 추가
