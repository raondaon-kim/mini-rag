# AI 교육 설계 (AIED) 가이드

## AIED 설계 프레임워크

### AI 교육 적용 유형
| 유형 | 설명 | 기술 |
|------|------|------|
| **AI 튜터** | 1:1 맞춤 피드백, 질의응답 | LLM, RAG |
| **적응형 학습** | 수준별 문제/경로 자동 조정 | 학습 분석, BKT |
| **자동 평가** | 서술형/코딩 자동 채점 | NLP, 코드 실행 |
| **콘텐츠 생성** | 문제 자동 출제, 설명 생성 | LLM, 지식그래프 |
| **학습 분석** | 학습 패턴, 예측, 개입 | LA, EDM |

### 학습자 모델 설계
```
학습자 프로필
├── 인지적 요소: 사전지식, 역량수준, 학습속도
├── 정의적 요소: 동기, 자기효능감, 학습 태도
├── 행동적 요소: 학습시간, 접속빈도, 완료율
└── 맥락적 요소: 학습환경, 기기, 시간대
```

## AI 튜터 시나리오 설계

### 대화 흐름 패턴
```
[학습자 질문] 
  → [의도 파악] 
  → [지식 수준 판단]
  → [응답 전략 선택]
     ├── 초급: 개념부터 설명 + 예시
     ├── 중급: 핵심 요점 + 응용 질문
     └── 고급: 심화 논의 + 비판적 사고 유도
  → [이해 확인 질문]
  → [피드백 + 다음 단계 안내]
```

### 프롬프트 설계 원칙
1. **소크라테스식**: 답을 직접 주지 않고 질문으로 유도
2. **비계(Scaffolding)**: 힌트를 단계적으로 제공
3. **메타인지 촉진**: "왜 그렇게 생각했어요?"
4. **오류 기반 학습**: 오답 분석 → 오개념 교정

## 적응형 학습 설계

### 지식 공간 모델 (Knowledge Space)
```
전제 조건 그래프:
  개념A → 개념B → 개념D
              → 개념C → 개념E
```

### 적응 규칙 엔진
| 조건 | 액션 |
|------|------|
| 정답률 > 80% (3문제 연속) | 다음 수준으로 이동 |
| 정답률 < 50% | 보충 콘텐츠 제공 |
| 응답시간 > 2× 평균 | 힌트 제공 |
| 같은 오류 2회 반복 | 오개념 교정 콘텐츠 |
| 3일 이상 미접속 | 리마인더 + 복습 문제 |

## 학습 데이터 분석 (LA)

### 핵심 지표
| 지표 | 계산 | 벤치마크 |
|------|------|---------|
| 과정 완료율 | 완료자/등록자 | 온라인: 15-30% |
| 콘텐츠 참여도 | 시청시간/전체시간 | >70% 양호 |
| 평가 통과율 | 통과자/응시자 | 70-85% 적정 |
| 학습 유지율 | 30일 후 재접속 | >40% 양호 |
| NPS | 추천 의향 | >50 우수 |

### 학습 분석 대시보드 구성
```
[전체 현황] 수강생 수, 완료율, 평균 점수
[학습 경로] 이탈 구간, 병목 지점
[개인 추적] 학습자별 진도, 취약 영역
[예측] 중도 이탈 위험 학습자
```

## 프로액티브 트리거
- AI 튜터가 항상 답을 줌 → 소크라테스식 전환 권고
- 적응 규칙 없이 선형 학습 → 분기/조건부 경로 설계
- 학습 데이터 수집 계획 없음 → LA 지표 정의 먼저
- 개인정보 고려 없음 → 학습 데이터 윤리/동의 확인
- 교수자 대시보드 없음 → 교수자용 분석 뷰 추가
