# 작업 학습 일지 (Work Journal)

## 목적

모든 작업 완료 후 **무엇을 했고, 어떤 방식을 적용했고, 결과가 어땠는지** 기록하여:
1. 같은 유형의 작업 반복 시 이전 경험을 참고
2. 효과적이었던 접근을 재사용
3. 문제가 있었던 부분을 개선

## 기록 타이밍

### 작업 시작 시
`query_work_journal`로 유사한 이전 작업을 검색합니다:
- 같은 유형의 문서를 만든 적이 있는가?
- 어떤 프레임워크를 적용했는가?
- 사용자 피드백이 있었는가?
- 수정/개선한 이력이 있는가?

### 작업 완료 후
`save_work_journal`로 기록합니다:

```
필수 항목:
- task_type: 작업 유형 (report, proposal, ppt, excel, search, analysis 등)
- skill_used: 적용한 Skill/프레임워크
- description: 수행한 작업 내용 요약
- output_file: 생성된 파일 경로 (있을 때)

선택 항목:
- user_feedback: 사용자 반응/피드백 (있을 때)
- lessons: 배운 점, 다음에 다르게 할 것
- quality_notes: 잘된 점, 개선할 점
```

## 이전 기록 활용 방법

이전 기록에서 다음을 참고합니다:

| 참고 항목 | 활용 방법 |
|-----------|----------|
| `skill_used` | 같은 유형에 이전에 효과적이었던 프레임워크 재사용 |
| `user_feedback` | 부정적 피드백이 있었으면 해당 부분 개선 |
| `lessons` | "다음에 다르게 할 것"을 이번에 적용 |
| `quality_notes` | 잘된 점은 유지, 개선점은 반영 |

## 예시 흐름

```
사용자: "월간 보고서 만들어줘"
  │
  ▼ query_work_journal({task_type: "report"})
  │
  ├─ 이전 기록 발견:
  │   "2주 전 월간 보고서 작성. pyramid-scqa 적용.
  │    사용자 피드백: '요약이 너무 길다. 3줄 이내로'
  │    lessons: '요약을 200자 이내로 제한할 것'"
  │
  ▼ 이번 작업에 적용:
  │   - pyramid-scqa 프레임워크 사용 (이전과 동일)
  │   - 요약을 200자 이내로 제한 (피드백 반영)
  │
  ▼ 작업 완료 후:
     save_work_journal({
       task_type: "report",
       skill_used: "pyramid-scqa",
       description: "3월 월간 보고서 작성",
       output_file: "data/output/3월_월간보고서.docx",
       lessons: "요약 200자 제한 적용 — 사용자 만족"
     })
```

## 피드백 감지

사용자의 다음 발언을 피드백으로 감지합니다:

| 사용자 발언 | 피드백 유형 | 기록 |
|------------|-----------|------|
| "좋아", "완벽해", "잘 됐어" | 긍정 | quality_notes에 긍정 기록 |
| "좀 짧게", "너무 길어" | 수정 요청 | lessons에 "분량 줄일 것" |
| "형식을 바꿔줘", "표로 만들어" | 형식 변경 | lessons에 선호 형식 기록 |
| "이전이랑 같은 스타일로" | 반복 요청 | 이전 기록의 설정을 그대로 적용 |
| "다르게 해줘", "이번엔 다른 방식" | 변경 요청 | 이전과 다른 Skill 선택 |
