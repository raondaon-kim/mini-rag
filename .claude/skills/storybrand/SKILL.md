# StoryBrand SB7 — Donald Miller

**출처**: Donald Miller, "Building a StoryBrand" (2017). 고객을 영웅으로, 브랜드를 가이드로 포지셔닝하는 메시징 프레임워크.

## 7요소 구조

### 1. 주인공 (Character) — 고객이 영웅이다
```
고객이 원하는 것을 한 문장으로 정의
- "재무 자유를 원하는 사업주"
- ⚠️ 브랜드가 영웅이 되면 안 됨
```

### 2. 문제 (Problem) — 3단계로 정의
```
- 외적 문제: 눈에 보이는 문제 ("보고서 작성에 12시간")
- 내적 문제: 감정적 고통 ("무능하다고 느끼는 것")
- 철학적 문제: 정의/당위성 ("이런 비효율은 있으면 안 된다")
```

### 3. 가이드 (Guide) — 브랜드 = 멘토
```
- 공감 먼저: "그 답답함, 저도 잘 압니다"
- 권위 그 다음: "10년간 500개 기업의 문제를 해결했습니다"
```

### 4. 계획 (Plan) — 3단계 간단 플랜
```
1단계: [간단한 행동] "무료 상담 신청"
2단계: [우리가 할 것] "맞춤 솔루션 설계"
3단계: [결과] "성과 달성"
```

### 5. 행동 촉구 (Call to Action)
```
- 직접 CTA: "지금 시작하기" (버튼)
- 전환 CTA: "무료 가이드 다운로드" (리드)
- 두 가지 모두 있어야 함
```

### 6. 실패 회피 (Failure)
```
"하지 않으면 잃는 것"을 명시
- 과장 없이, 사실 기반으로
```

### 7. 성공 (Success)
```
변화된 삶을 구체적으로 그리기
- "매주 12시간을 되찾아 가족과 보냅니다"
```

## 핵심 원칙
> **고객이 영웅, 브랜드가 가이드.** 브랜드가 자기 이야기를 하는 순간 고객은 떠난다.

## 품질 체크
- [ ] 모든 문장에서 고객이 주어인가? (브랜드가 주어 X)
- [ ] 문제가 3단계(외적/내적/철학적)로 정의되었는가?
- [ ] 가이드가 공감→권위 순서인가?
- [ ] 계획이 정확히 3단계인가?
- [ ] 직접 CTA + 전환 CTA 둘 다 있는가?

## 프로액티브 트리거
- 브랜드가 영웅으로 포지셔닝됨 (고객이 영웅이어야 함)
- 빌런/문제가 불명확하거나 3단계(외적/내적/철학적) 정의 누락
- 계획 단계가 너무 복잡함 (3단계 간단 플랜으로 축소 필요)
- 실패 시나리오가 누락됨 ("하지 않으면 잃는 것" 미제시)
- 가이드(브랜드)의 공감 표현이 없이 권위만 내세움

## 산출물

- 엘리베이터 피치 스크립트 (30초/60초 버전)
