# 품질관리 가이드

## QMS (Quality Management System) 프레임워크

### PDCA 사이클 (ISO 9001 기반)
```
Plan → Do → Check → Act
  │      │      │      │
  │      │      │      └─ 표준화/개선 조치
  │      │      └─ 성과 측정/감사
  │      └─ 실행/교육/기록
  └─ 목표/프로세스/자원 계획
```

### 품질 목표 설정
| 영역 | 지표 | 현재값 | 목표값 | 기한 |
|------|------|--------|--------|------|
| 제품 품질 | 불량률 | | <_% | |
| 고객 만족 | CSAT/NPS | | >_점 | |
| 프로세스 | 리드타임 | | <_일 | |
| 개선 | CAPA 완료율 | | >95% | |

## CAPA (Corrective & Preventive Action)

### CAPA 보고서 구조
| 항목 | 내용 |
|------|------|
| CAPA 번호 | CAPA-YYYY-NNN |
| 발생일/보고일 | |
| 유형 | 시정(Corrective) / 예방(Preventive) |
| 문제 설명 | 5W1H (무엇, 어디서, 언제, 누가, 왜, 어떻게) |
| 근본 원인 | 5-Why 또는 Fishbone 분석 |
| 즉시 조치 | 응급 대응 |
| 시정/예방 조치 | 근본 원인 해결 |
| 효과성 검증 | 검증 방법 + 기한 |
| 완료일/승인 | |

### 5-Why 분석
```
문제: [현상]
Why 1: ___?
Why 2: ___?
Why 3: ___?
Why 4: ___?
Why 5: ___? ← 근본 원인
```

## 프로세스 개선 (Lean/Six Sigma)

### DMAIC
1. **Define**: 문제 정의, 프로젝트 범위, CTQ (Critical to Quality)
2. **Measure**: 현재 성과 측정, 데이터 수집
3. **Analyze**: 근본 원인 분석, 데이터 분석
4. **Improve**: 개선안 실행, 파일럿
5. **Control**: 표준화, 모니터링 체계

- [ ] 효과성 검증 기한이 설정되었는가?
- [ ] 관련 문서가 최신 상태인가?

## 프로액티브 트리거
- 품질 목표가 SMART(구체적/측정가능/달성가능/관련성/기한)가 아닐 때 → SMART 형식으로 재정의
- CAPA에 근본 원인 분석(5-Why/Fishbone)이 없을 때 → 근본 원인 분석 수행 후 시정 조치
- 효과성 검증 기한이 미설정일 때 → 검증 방법 + 기한 명시
- 관련 문서(절차서/표준서)가 최신 버전이 아닐 때 → 문서 갱신 후 변경 이력 기록
- 교육 기록이 없을 때 → 관련 인원 교육 실시 + 기록 보관 체계 수립
