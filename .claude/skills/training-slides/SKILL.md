# Training Slides — Gagne's 9 Events + Assertion-Evidence

**출처**: Robert Gagne, 9 Events of Instruction (1965, 교수설계 분야 표준) + Michael Alley, Assertion-Evidence 슬라이드 디자인 (Penn State)

## 슬라이드 흐름 (Gagne's 9 Events)

| # | 이벤트 | 슬라이드 내용 | 예시 |
|---|--------|-------------|------|
| 1 | 주의 끌기 | 놀라운 통계, 질문, 사례 | "여러분의 80%는 이걸 잘못 알고 있습니다" |
| 2 | 목표 제시 | 이 교육 후 할 수 있는 것 | "이 세션을 마치면 ~할 수 있습니다" |
| 3 | 사전 지식 연결 | 이미 아는 것과 연결 | "지난 주 배운 X를 기반으로..." |
| 4 | 새 내용 제시 | 핵심 개념 설명 | 개념별 1-2 슬라이드 |
| 5 | 학습 안내 | 예시, 비유, 다이어그램 | 시각적 설명 |
| 6 | 연습 유도 | 퀴즈, 실습, 토론 질문 | "직접 해봅시다" |
| 7 | 피드백 제공 | 정답/해설, 일반적 오류 | "많이 틀리는 부분은..." |
| 8 | 성과 평가 | 요약 퀴즈, 체크리스트 | "5가지를 모두 답할 수 있나요?" |
| 9 | 기억 강화 | 핵심 요약 + 실무 적용 팁 | "내일부터 적용할 3가지" |

## Assertion-Evidence 슬라이드 디자인

```
┌─────────────────────────────────────────┐
│ [주장 문장 — 완전한 한 문장, 2줄 이내]    │
│                                         │
│         [시각적 증거]                     │
│    다이어그램 / 예시 / 코드 / 이미지       │
│                                         │
└─────────────────────────────────────────┘
```

- 제목 = **주장**(assertion), 토픽(X)
- 본문 = **시각적 증거**(evidence)로 주장 뒷받침
- 불릿 포인트 최소화 → 다이어그램/이미지로 대체

## 규칙
- 학습 목표를 Bloom's Taxonomy 동사로: 이해하다, 적용하다, 분석하다, 평가하다, 창조하다
- 15분마다 인터랙션 삽입 (질문, 퀴즈, 실습)
- 슬라이드당 1개 개념만
- 발표자 노트에 상세 설명 + 예시 추가

## 프로액티브 트리거
- 학습목표 없이 내용만 나열됨 (Bloom's Taxonomy 동사 필요)
- 슬라이드당 텍스트 과다 (5줄 이상, 시각적 증거로 대체 필요)
- 활동/실습 슬라이드가 없음 (15분마다 인터랙션 필요)
- 요약/복습 슬라이드가 누락됨 (Gagne 8-9단계)
- 사전 지식 연결 없이 바로 새 내용으로 진입
- Assertion-Evidence 형식 미준수 (토픽형 제목 사용)

## 산출물
- Gagne 9단계 교육 PPTX (학습목표 + 실습 포함)
- 발표자 노트 (상세 설명 + 예상 질문/답변)
- 학습자 핸드아웃/워크시트
- 품질 체크 완료 리포트
