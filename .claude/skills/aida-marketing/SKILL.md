# AIDA — Attention, Interest, Desire, Action

**출처**: E. St. Elmo Lewis (1898). 마케팅 역사상 가장 오래되고 검증된 프레임워크.

**적용 대상**: 세일즈 페이지, 이메일 마케팅, 광고 카피, 제품 설명, 모금 어필

## 구조

### A — Attention (주의 끌기)
```
- 대담한 헤드라인, 놀라운 통계, 도발적 질문
- "스크롤을 멈추게 하는가?" 테스트 통과해야 함
- 예: "당신의 팀이 매주 12시간을 낭비하고 있다면?"
```

### I — Interest (흥미 유발)
```
- 독자의 상황/고통에 공감
- "이것은 나에 대한 이야기다" 느낌을 줘야 함
- 사실, 이야기, 공감할 수 있는 시나리오 활용
```

### D — Desire (욕구 자극)
```
- 기능(feature)이 아닌 혜택(benefit)에 집중
- After-state를 생생하게 그리기
- 증거: 고객 후기, 데이터, 사례 연구
- 반론 처리: "하지만 비용이..." → 미리 답변
```

### A — Action (행동 촉구)
```
- 단일, 구체적, 즉각적 CTA
- "더 알아보기"(X) → "무료 체험 시작하기"(O)
- 긴급성 또는 희소성 (진실한 경우만)
- 리스크 제거: "30일 환불 보장", "카드 정보 불필요"
```

## 품질 체크
- [ ] 헤드라인이 스크롤을 멈추게 하는가?
- [ ] Interest가 독자의 구체적 고통점을 다루는가?
- [ ] Desire가 기능이 아닌 혜택에 집중하는가?
- [ ] 증거(후기/데이터) 최소 1개 포함?
- [ ] CTA가 단일하고 구체적인가?

## 프로액티브 트리거
- Attention이 제목만 있고 본문 Hook이 없음
- Interest가 기능 나열에 그침 (독자 상황/공감 부족)
- Desire에 사회적 증거(후기/데이터/사례)가 부족함
- Action CTA가 모호함 ("더 알아보기" 수준)
- 단일 채널만 고려하고 멀티채널 전략이 없음
- Attention→Action 흐름이 논리적으로 연결되지 않음

## 산출물
- AIDA 구조 마케팅 카피 (헤드라인 + 본문 + CTA)
- 채널별 변형본 (이메일/SNS/랜딩페이지)
- A/B 테스트용 헤드라인 후보 3-5개
- 품질 체크 완료 리포트
