# HRD 교육 설계

## ADDIE 모델 (교수설계 표준)

### 1. 분석 (Analyze)
```
- 학습 대상: [누구?] 기존 지식 수준: [어느 정도?]
- 성과 격차: [현재 vs 기대 수준]
- 제약 조건: [시간, 예산, 기술 환경]
- 학습 목표: [측정 가능한 Bloom 동사로]
```

### 2. 설계 (Design)
```
- 학습 목표 (Bloom's Taxonomy):
  "이 과정을 마치면 학습자는 [동사] + [내용]을 할 수 있다"
  
  기억: 정의하다, 나열하다, 식별하다
  이해: 설명하다, 요약하다, 비교하다
  적용: 실행하다, 사용하다, 해결하다
  분석: 구분하다, 비교하다, 추론하다
  평가: 판단하다, 비판하다, 검증하다
  창조: 설계하다, 개발하다, 구성하다

- 평가 방법: [퀴즈/실습/프로젝트]
- 콘텐츠 순서: 쉬운것 → 어려운것 (스캐폴딩)
```

### 3. 개발 (Develop) — 콘텐츠 유형
| 유형 | 포맷 | 적합한 학습 |
|------|------|-----------|
| 강의 슬라이드 | PPT (training-slides Skill) | 개념 전달 |
| 실습 워크시트 | Excel/PDF | 연습/적용 |
| 퀴즈 | 객관식/주관식 | 확인/평가 |
| 사례 연구 | DOCX/PDF | 분석/종합 |
| 체크리스트 | DOCX | 절차 확인 |

### 4. 실행 (Implement) + 5. 평가 (Evaluate)
- Kirkpatrick 4단계: 반응 → 학습 → 행동 → 결과

## 퀴즈 생성 규칙

### 객관식
```
Q: [Bloom 수준에 맞는 질문]
A) [정답] ← 명확히 맞는 것
B) [오답 - 그럴듯하지만 틀린 것]
C) [오답 - 흔한 오해]
D) [오답 - 부분적으로만 맞는 것]
```

### 주관식
```
Q: [서술형 질문 — "설명하시오", "비교하시오"]
모범답안: [핵심 키워드 3-5개 포함]
채점 기준: [키워드 포함 여부 + 논리적 구조]
```

## 온보딩 체크리스트

```
## 입사 전
- [ ] 장비 준비 (노트북, 계정)
- [ ] 환영 이메일 발송
- [ ] 멘토 배정

## 첫째 주
- [ ] 회사 소개 및 조직 구조
- [ ] 핵심 시스템 접근 권한
- [ ] 팀 소개 미팅
- [ ] 업무 도구 교육

## 첫째 달
- [ ] 핵심 업무 프로세스 학습
- [ ] 첫 프로젝트 참여
- [ ] 1:1 미팅 (매주)
- [ ] 30일 피드백 세션
```

## 프로액티브 트리거
- 학습목표가 "~이해한다"처럼 관찰 불가능한 동사 사용 → Bloom 동사로 교체
- 평가 방법이 학습목표와 정렬되지 않음 → 목표별 평가 매핑 확인
- 강의만 있고 실습/활동이 없음 → 능동적 학습 활동 추가
- 온보딩 체크리스트에 기한/담당자 없음 → 각 항목에 기한+담당 추가
- 교육 효과 측정 계획 없음 → Kirkpatrick 4단계 중 최소 2단계 설계
