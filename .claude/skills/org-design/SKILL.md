# 조직 설계 가이드

## 조직 구조 유형

| 유형 | 특징 | 적합 상황 |
|------|------|----------|
| 기능별 | 마케팅팀, 개발팀 등 전문 기능 | 안정적, 전문성 중시 |
| 사업부별 | 제품/지역/고객별 독립 부서 | 다각화, 빠른 의사결정 |
| 매트릭스 | 기능 × 프로젝트 교차 | 복합 프로젝트 |
| 플랫 | 계층 최소화, 자율 | 스타트업, 소규모 |
| 네트워크 | 핵심 + 외부 파트너 | 유연성, 확장성 |

## 역할 정의 (R&R) 프레임워크

### RACI 매트릭스
| 업무/프로세스 | 팀A | 팀B | 팀C | 임원 |
|-------------|-----|-----|-----|------|
| 기획 | R | C | I | A |
| 실행 | C | R | I | I |
| 승인 | I | I | C | A |

- **R**esponsible: 실행 담당
- **A**ccountable: 최종 책임 (1명만)
- **C**onsulted: 자문/협의
- **I**nformed: 결과 통보

### 직무 기술서 (Job Description) 구조
| 항목 | 내용 |
|------|------|
| 직책명 | |
| 소속 | 부서/팀 |
| 보고 라인 | 상위 직책 |
| 핵심 책임 | 3-5개 주요 업무 |
| 성과 지표 | KPI 2-3개 |
| 필요 역량 | 스킬, 경험, 자격 |

## 조직 진단 도구

### 맥킨지 7S
| 요소 | Hard/Soft | 점검 항목 |
|------|-----------|---------|
| Strategy | Hard | 전략 방향 명확한가? |
| Structure | Hard | 구조가 전략을 지원하는가? |
| Systems | Hard | 프로세스/도구가 적절한가? |
| Skills | Soft | 필요 역량을 보유했는가? |
| Staff | Soft | 적재적소 배치인가? |
| Style | Soft | 리더십 스타일이 적합한가? |
| Shared Values | Soft | 공유 가치가 정렬되었는가? |

## 조직 변경 시 체크리스트
- [ ] 변경 목적이 전략과 연결되는가?
- [ ] 영향 받는 인원에게 사전 소통했는가?
- [ ] R&R이 명확히 재정의되었는가?
- [ ] 보고 라인이 깔끔한가? (1명 → 1상위자)
- [ ] 전환 기간 계획이 있는가?

## 프로액티브 트리거
- 전략과 조직 구조의 연결이 부족함
- RACI에서 A(Accountable)가 2명 이상임
- 보고 라인이 교차함 (1인 2상위자)
- R&R에 중복 또는 공백이 있음
- 전환 기간 계획이 없음
