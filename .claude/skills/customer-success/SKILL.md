# 고객 성공 (Customer Success) 가이드

## 고객 여정 단계

```
가입 → 온보딩 → 활성화 → 성장 → 갱신 → 확장
                                    ↓ (위험 시)
                                  이탈 방지
```

### 단계별 핵심 활동
| 단계 | 활동 | 성공 지표 |
|------|------|----------|
| 온보딩 | 초기 설정, 교육, 첫 성공 경험 | Time-to-Value |
| 활성화 | 핵심 기능 사용, 팀 확대 | DAU/MAU, 기능 채택률 |
| 성장 | 고급 기능, 베스트 프랙티스 | NPS, 사용량 증가 |
| 갱신 | 성과 리뷰, ROI 입증 | 갱신율 |
| 확장 | 업셀/크로스셀 | 확장 MRR |

## 고객 건강도 스코어

| 신호 | 가중치 | 측정 방법 |
|------|--------|---------|
| 제품 사용량 | 30% | 로그인, 핵심 기능 사용 |
| 참여도 | 25% | 미팅 참석, 응답 속도 |
| 지원 이력 | 20% | 티켓 수, 심각도 |
| 계약 상태 | 15% | 잔여 기간, 결제 이력 |
| 성장 신호 | 10% | 사용자 추가, 기능 문의 |

**건강도 등급**:
- 🟢 80-100: 건강 → 확장 기회 탐색
- 🟡 50-79: 주의 → 선제적 관리
- 🔴 0-49: 위험 → 즉시 개입

## 성과 리뷰 (QBR) 템플릿
1. **달성 성과**: 목표 대비 실적
2. **ROI 입증**: 비용 절감, 생산성 향상 수치화
3. **사용 현황**: 기능별 채택률
4. **개선 제안**: 미활용 기능, 베스트 프랙티스
5. **다음 분기 목표**: 합의된 성공 지표

## 이탈 방지 플레이북
| 이탈 신호 | 대응 | 타이밍 |
|----------|------|--------|
| 사용량 급감 | 1:1 체크인, 교육 제안 | 즉시 |
| 핵심 담당자 퇴사 | 새 담당자 온보딩 | 1주 내 |
| 지원 불만 접수 | 에스컬레이션, 경영진 참여 | 24시간 내 |
| 갱신 3개월 전 | 성과 리뷰, ROI 보고 | D-90 |

## 프로액티브 트리거
- 고객 건강도 점수가 정의되지 않았을 때 → 가중치 기반 건강도 스코어링 모델 설계
- QBR(분기 리뷰)가 없을 때 → QBR 템플릿 + 일정 수립
- 이탈 신호 모니터링이 없을 때 → 이탈 신호 목록 + 자동 알림 기준 설정
- 온보딩 완료 기준이 없을 때 → Time-to-Value 기준점 + 체크리스트 정의
- 확장(업셀/크로스셀) 기회가 추적되지 않을 때 → 확장 파이프라인 추적 체계 수립
