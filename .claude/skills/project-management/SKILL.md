# 프로젝트 관리

## WBS (Work Breakdown Structure)

```
# 프로젝트: [프로젝트명]

## 1. [대분류 1]
  ### 1.1 [중분류]
    - 1.1.1 [작업] — 담당: [이름] — 기간: [X일] — 선행: [없음]
    - 1.1.2 [작업] — 담당: [이름] — 기간: [X일] — 선행: [1.1.1]
  ### 1.2 [중분류]
    - 1.2.1 [작업]

## 2. [대분류 2]
  ### 2.1 [중분류]
    - 2.1.1 [작업]
```

## 리스크 매트릭스

```
| ID | 리스크 | 확률 | 영향 | 등급 | 완화 방안 | 담당 |
|----|--------|------|------|------|----------|------|
| R1 | [리스크] | 높음 | 높음 | 🔴 | [방안] | [이름] |
| R2 | [리스크] | 중간 | 높음 | 🟡 | [방안] | [이름] |
| R3 | [리스크] | 낮음 | 낮음 | 🟢 | 모니터링 | [이름] |
```

확률×영향 매트릭스:
```
         낮음    중간    높음
높음      🟡     🟡     🔴
중간      🟢     🟡     🟡
낮음      🟢     🟢     🟡
```

## 스프린트 계획

```
## Sprint [번호]: [목표]
- 기간: [시작일] ~ [종료일]
- 스프린트 목표: [한 문장]

### 백로그
| 우선순위 | 작업 | 스토리 포인트 | 담당 | 상태 |
|---------|------|-------------|------|------|
| P0 | [작업] | 5 | [이름] | ⬜ |
| P1 | [작업] | 3 | [이름] | ⬜ |
| P2 | [작업] | 2 | [이름] | ⬜ |

### 총 SP: [합계] / 팀 용량: [X SP]
```

## 회고 (Retrospective)

```
## Sprint [번호] 회고

### 😊 잘한 것 (Keep)
- [...]

### 😐 개선할 것 (Problem)
- [...]

### 💡 시도할 것 (Try)
- [...] — 담당: [이름] — 기한: [다음 스프린트]
```

## 프로액티브 트리거
- WBS 최하위 작업이 2주를 초과함
- 리스크 매트릭스가 없음
- 마일스톤 없는 일정이 나열되고 있음
- 담당자가 미지정된 작업이 있음
- 작업 간 의존관계가 식별되지 않음
