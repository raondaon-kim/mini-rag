# IMRaD — Introduction, Methods, Results, Discussion

**출처**: ICMJE (International Committee of Medical Journal Editors). 1970년대부터 과학 분야 표준. 전 세계 학술지의 지배적 구조.

**적용 대상**: 연구 논문, 실험 보고서, 학술 발표, 데이터 분석 보고

## 구조

### 1. 제목 & 초록 (Abstract)
- 목적, 방법, 핵심 발견, 결론 (150-300 단어)
- **초록은 모든 섹션을 다 쓴 후 마지막에 작성**

### 2. 서론 (Introduction) — "왜 이 연구를 했는가?"
```
넓은 맥락 → 좁은 연구 질문 (깔때기 구조)
- 배경과 맥락 [선행 연구 인용]
- 연구 격차/문제점 [아직 밝혀지지 않은 것]
- 연구 목적/가설 [마지막 문장]
```

### 3. 방법 (Methods) — "어떻게 조사했는가?"
```
- 연구 대상/데이터: [표본 크기, 선정 기준]
- 도구/측정: [사용한 도구, 측정 방법]
- 절차: [단계별 과정]
- 분석 방법: [통계 기법, 소프트웨어]
```

### 4. 결과 (Results) — "무엇을 발견했는가?"
```
- 데이터만 제시, 해석 없음
- 표/그림으로 핵심 데이터 제시
- 모든 표/그림은 본문에서 참조
- 통계적 유의성 보고 (p값, 신뢰구간)
```

### 5. 논의 (Discussion) — "이것이 무엇을 의미하는가?"
```
- 첫 문단: 핵심 발견 재진술
- 선행 연구와 비교 [일치/불일치]
- 한계점 [솔직하게]
- 시사점 [실무적/이론적 의미]
- 향후 연구 방향
```

## 품질 체크
- [ ] 초록만 읽고도 연구 전체를 파악할 수 있는가?
- [ ] 서론이 깔때기 구조(넓음→좁음)인가?
- [ ] 방법이 재현 가능할 만큼 상세한가?
- [ ] 결과에 해석이 섞이지 않았는가?
- [ ] 논의의 첫 문단이 핵심 발견으로 시작하는가?
- [ ] 한계점이 솔직하게 다뤄졌는가?

## 프로액티브 트리거
- 연구 질문이 불명확하거나 누락되었을 때 → 서론 마지막에 연구 질문/가설 명시
- Introduction에 선행연구 리뷰가 부족할 때 → 관련 선행연구 인용 추가
- Methods 섹션이 재현 불가능할 정도로 불충분할 때 → 표본, 도구, 절차 상세화
- Results와 Discussion이 혼재되어 있을 때 → 데이터 제시(Results)와 해석(Discussion) 분리
- 참고문헌 형식이 불일치할 때 → 단일 인용 스타일(APA/Vancouver 등)로 통일
- 초록이 본문 작성 전에 쓰여졌을 때 → 본문 완성 후 초록 재작성 권고
