# 로드맵 가이드

## 로드맵 유형

| 유형 | 대상 | 상세도 | 기간 |
|------|------|--------|------|
| 비전 로드맵 | 경영진, 이사회 | 테마 수준 | 1-3년 |
| 전략 로드맵 | 부서장, PM | 기능 수준 | 6-12개월 |
| 릴리즈 로드맵 | 개발팀 | 태스크 수준 | 1-3개월 |

## 테마 기반 로드맵 구조

### 타임라인
```
        Now         Next          Later
       (Q2)        (Q3)          (Q4+)
    ┌─────────┐ ┌─────────┐ ┌─────────┐
    │ 테마 1   │ │ 테마 3   │ │ 테마 5   │
    │ 테마 2   │ │ 테마 4   │ │ 테마 6   │
    └─────────┘ └─────────┘ └─────────┘
    확정/진행중    계획됨       탐색 중
```

### 마일스톤 정의
| 마일스톤 | 목표일 | 핵심 결과물 | 의존성 | 상태 |
|---------|--------|-----------|--------|------|
| M1 | | | | |
| M2 | | | | |

### Now / Next / Later 프레임워크
- **Now**: 확정, 리소스 할당, 상세 스펙 완료
- **Next**: 방향 합의, 대략적 범위, 리소스 계획 중
- **Later**: 비전/탐색 단계, 변경 가능

## 로드맵 엑셀/PPT 구성
- **행**: 제품 영역 또는 팀
- **열**: 시간 (월/분기)
- **셀**: 기능/이니셔티브 (색상으로 상태 구분)
- **범례**: 초록=완료, 파랑=진행, 노랑=계획, 회색=탐색

## 체크리스트
- [ ] 전략 목표와 연결되었는가?
- [ ] 의존성이 식별되었는가?
- [ ] 리소스 제약이 반영되었는가?
- [ ] 이해관계자 리뷰를 거쳤는가?

## 프로액티브 트리거
- Now/Next/Later 구분 없이 전부 "확정"으로 표시할 때 → 확정도에 따라 3단계 분류
- 의존성(Dependency)이 식별되지 않았을 때 → 기능/팀 간 의존성 맵 작성
- 리소스 제약이 반영되지 않았을 때 → 팀 가용성 대비 로드 검증
- 전략 목표와 로드맵 항목이 연결되지 않았을 때 → 각 이니셔티브에 전략 목표 태그
- 이해관계자 리뷰 없이 확정할 때 → 리뷰 프로세스 포함
