# BLUF — Bottom Line Up Front

**출처**: 미군 규정 AR 25-50 (Preparing and Managing Correspondence). 군사/정부 커뮤니케이션 필수 규정. Fortune 500 기업에서 널리 채택.

**적용 대상**: 이메일, 짧은 메모, 상태 보고, Slack/Teams 메시지, 경영진 대상 모든 커뮤니케이션

---

## 구조

```
BLUF: [독자가 알아야 할 가장 중요한 한 가지. 첫 문장.]

배경: [필요한 경우에만 맥락 설명]
상세: [뒷받침 정보, 데이터]
요청 사항: [독자에게 필요한 구체적 행동]
기한: [언제까지]
```

## 예시

### 승인 요청 이메일

```
BLUF: 마케팅 예산 15% 증액 ($50K → $57.5K) 승인을 요청합니다.

배경: Q3 디지털 광고 성과가 ROAS 4.2x로 목표(3.0x) 대비 40% 초과 달성했습니다.
상세: 추가 $7.5K를 Facebook/Google 리타겟팅에 집중 투자하면, Q4 매출 $30K 추가 전환이 예상됩니다.
요청: 10/15까지 예산 증액 승인 부탁드립니다.
```

### 상태 보고

```
BLUF: 프로젝트 Alpha는 예정대로 11/30 런칭 가능합니다.

완료: 백엔드 API (100%), 프론트엔드 UI (90%)
진행 중: QA 테스트 (60%, 11/20 완료 예정)
리스크: 결제 모듈 외부 API 응답 지연 — 대안 준비 중
필요 사항: 없음 (정보 공유 목적)
```

## 핵심 원리

> **첫 문장에서 결론을 말하라.** 바쁜 독자는 첫 문장만 읽을 수 있다. 그 한 문장이 전체 메시지를 전달해야 한다.

## 변형: SCR (짧은 보고용)

```
S (Situation): [현재 상태 — 1문장]
C (Complication): [문제/변화 — 1문장]
R (Resolution): [해결/제안 — 1문장]
```

## 품질 체크

- [ ] 첫 문장만 읽어도 핵심을 파악할 수 있는가?
- [ ] 요청 사항이 구체적인가? (누가, 무엇을, 언제)
- [ ] 전체 길이가 스크롤 없이 읽히는가?
- [ ] 불필요한 배경 설명이 제거되었는가?

## 프로액티브 트리거

- 결론이 3문장을 초과하면 → 1문장 BLUF로 압축 권고
- 배경 설명이 결론보다 앞에 위치하면 → BLUF를 첫 문장으로 이동 제안
- 액션 아이템이 불명확하면 → "누가, 무엇을, 언제"로 구체화 요구
- 이메일/메모가 스크롤해야 읽히는 길이이면 → 핵심만 남기고 축약 유도
- "참고로", "배경을 설명하면" 등으로 시작하면 → 결론 선행 구조로 재배치 권고
- 요청 사항이 없는 정보 공유와 승인 요청이 구분되지 않으면 → 목적 명시 유도

## 산출물

- BLUF 구조의 이메일/메모/상태 보고 (텍스트/Markdown)
- 결론 선행형 경영진 커뮤니케이션
- SCR 변형의 짧은 보고서
- 액션 아이템 포함 의사결정 요청 메시지
