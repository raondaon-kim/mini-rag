# Status Report PPT — SCR 프레임워크

**출처**: McKinsey SCR(Situation-Complication-Resolution) + 표준 PMO 보고 형식

## 5-8장 구조

| # | 슬라이드 | 내용 |
|---|---------|------|
| 1 | 표지 | 보고 기간, 프로젝트명, 작성자 |
| 2 | 한 눈 요약 | 신호등(🟢🟡🔴) + 핵심 KPI 3개 |
| 3 | 완료된 것 | 이번 기간 달성 사항 (체크리스트) |
| 4 | 진행 중 | 현재 작업 + 완료율 + 예상 일정 |
| 5 | 이슈/리스크 | 빨간불 항목 + 완화 방안 |
| 6 | 다음 기간 계획 | 마일스톤 + 담당자 |
| 7 | 요청 사항 | 필요한 의사결정/지원 (있을 때만) |

## 신호등 시스템
- 🟢 **Green**: 계획대로 진행
- 🟡 **Yellow**: 주의 필요, 완화 중
- 🔴 **Red**: 즉시 조치 필요

## 규칙
- 첫 슬라이드에서 전체 상태를 즉시 파악 가능해야 함
- 이슈는 문제 + 원인 + 완화 방안을 함께 제시
- 모든 행동 항목에 담당자 + 기한 필수

## 프로액티브 트리거
- 데이터 없이 정성적 서술만 있음 (수치/KPI 필요)
- 비교 기준(전주/전월/목표)이 없어 진척 판단 불가
- 이슈만 있고 해결방안/완화계획이 없음
- 슬라이드가 글자로 빽빽함 (신호등/차트로 시각화 필요)
- 담당자/기한 없는 행동 항목이 존재
- 한 눈 요약 슬라이드가 누락됨

## 산출물
- 5-8장 상태보고 PPTX (SCR 프레임워크)
- 신호등 대시보드 슬라이드 (KPI 3개 + 상태)
- 이슈/리스크 매트릭스 (문제 + 원인 + 완화방안)
- 품질 체크 완료 리포트
