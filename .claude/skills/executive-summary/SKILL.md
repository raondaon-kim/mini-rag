# Executive Summary Blueprint

**출처**: Harvard Business Review 스타일 가이드, Fortune 500 기업 표준 관행의 종합.

**적용 대상**: 보고서 요약, 경영진 브리핑, 투자 메모 요약, 의사결정 지원 문서

---

## 구조

```
1. 목적 (1-2문장)
   이 문서의 주제와 작성 이유. 어떤 의사결정을 지원하는지.

2. 핵심 발견/권고 (1-3문장)
   가장 중요한 결론. 수치 포함. 헤지 없이 직접적으로.

3. 주요 근거 (3-5개 불릿)
   각 불릿에 핵심 데이터 포인트 포함.
   - [근거 1]: [데이터]
   - [근거 2]: [데이터]
   - [근거 3]: [데이터]

4. 영향/시사점 (1-2문장)
   조직에 미치는 재무적/전략적 의미.

5. 권고 다음 단계 (2-4개 불릿)
   - [행동] — 담당: [이름] — 기한: [날짜]
   - [행동] — 담당: [이름] — 기한: [날짜]
```

## 규칙

| 항목 | 규칙 |
|------|------|
| 분량 | 원문의 10% 이내 (10페이지 보고서 → 1페이지 요약) |
| 작성 순서 | 본문을 다 쓴 후 마지막에 작성 |
| 배치 | 문서의 맨 앞에 배치 |
| 독립성 | 이것만 읽고도 올바른 의사결정 가능해야 함 |
| 수치 | 읽기 쉽게 반올림 ($4,237,892 → ~$4.2M) |
| 톤 | 자신 있게, 직접적으로. "~일 수 있다" 대신 "~이다" |

## 품질 체크

- [ ] 이 요약만 읽고도 올바른 의사결정이 가능한가?
- [ ] 권고안이 첫 문단에 있는가?
- [ ] 핵심 포인트가 5개 이하인가?
- [ ] 모든 수치가 반올림되어 읽기 쉬운가?
- [ ] 전문 용어 없이 이해 가능한가?
- [ ] Call to Action이 명확한가? (누가, 무엇을, 언제)
- [ ] 200-500단어 이내인가?

## 프로액티브 트리거

- 요약이 원문의 30%를 초과하면 → 10% 이내로 압축 권고
- 원문에 없는 새로운 정보가 요약에 포함되면 → 해당 내용 제거 또는 본문 이동 제안
- 핵심 수치나 결론이 요약에서 누락되면 → 의사결정에 필수적인 데이터 포인트 추가 요구
- 권고안이 첫 문단에 없으면 → 핵심 발견/권고를 서두로 이동 유도
- 전문 용어가 설명 없이 사용되면 → 경영진 대상 평이한 언어로 변환 권고
- 다음 단계에 담당자/기한이 빠져 있으면 → 실행 가능한 Action Items로 보완 요청

## 산출물

- 200-500단어의 Executive Summary (Markdown/DOCX)
- 핵심 발견 + 권고안 + 다음 단계 포함 의사결정 지원 문서
- 원문 대비 10% 이내의 압축 브리핑
- 독립적으로 읽을 수 있는 경영진 보고 요약
