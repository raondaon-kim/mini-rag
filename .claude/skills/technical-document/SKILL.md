# Technical Document — 역피라미드 + IEEE 표준

**출처**: IEEE Std 1063 (소프트웨어 사용자 문서 표준) + ISO/IEC 26514 (기술 문서 표준) + 역피라미드 기법(저널리즘)

## 표준 구조

```
1. 제목 페이지
   - 문서 제목, 버전, 날짜, 작성자, 승인자

2. 변경 이력 (2+ 버전부터)
   | 버전 | 날짜 | 변경 내용 | 작성자 |

3. 목차 (5페이지 이상일 때)

4. 요약 (문서의 10% 이내)
   - 이것만 읽고도 핵심 파악 가능

5. 서론
   - 목적, 범위, 대상 독자, 용어 정의

6. 본문 (섹션별)
   각 섹션: H2 → H3 → 본문 → 표/그림
   논리 흐름: 개요 → 상세 → 예시 → 주의사항

7. 결론 및 권고
   - 핵심 발견 번호 매기기
   - 구체적 권고 사항

8. 부록 (선택)
   - 참조 데이터, 용어집, 상세 스펙
```

## 제목 체계

| 수준 | 용도 | 번호 체계 |
|------|------|----------|
| H1 | 대단원 | 1. 2. 3. |
| H2 | 중단원 | 1.1 1.2 2.1 |
| H3 | 소단원 | 1.1.1 1.1.2 |
| 본문 | 일반 텍스트 | (번호 없음) |

## 기술 문서 특유 규칙

- **능동태 우선**: "분석이 수행되었다" → "팀이 분석했다"
- **전문 용어**: 첫 등장 시 정의, 이후 약어 가능 — "RAG(Retrieval-Augmented Generation, 이하 RAG)"
- **표/그림**: 번호 필수, 표 제목=위, 그림 제목=아래
- **코드/명령**: 고정폭 폰트, 별도 블록
- **경고/주의**: ⚠️ 접두사로 시각적 구분
- **버전 관리**: 변경 이력 표 필수

## 프로액티브 트리거
- 대상 독자 수준이 정의되지 않았을 때 → 독자 수준 확인 후 용어/설명 깊이 조정
- 용어 정의 없이 전문 용어를 사용할 때 → 첫 등장 시 정의 추가
- 코드 예시 없이 API를 설명할 때 → 최소 1개 코드 예시 삽입
- 버전/날짜가 기재되지 않았을 때 → 제목 페이지에 버전, 날짜, 작성자 추가
- 목차/색인 없이 5페이지 이상일 때 → 자동 목차 생성
- 변경 이력 표가 누락되었을 때 → 변경 이력 섹션 추가
