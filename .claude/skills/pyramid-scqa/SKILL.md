# Minto Pyramid Principle + SCQA

**출처**: Barbara Minto, McKinsey & Company (1973). 50년간 전 세계 컨설팅 펌에서 검증된 글쓰기의 골드 스탠다드.

**적용 대상**: 비즈니스 보고서, 컨설팅 보고서, 전략 분석, 이사회 자료, 투자 메모

---

## 서론: SCQA 프레임워크

모든 보고서의 서론은 이 4단계로 시작합니다:

| 단계 | 역할 | 분량 |
|------|------|------|
| **S** (Situation) | 독자가 이미 아는 안정적 배경 | 1-3문장 |
| **C** (Complication) | 문제, 변화, 긴장 요소 | 1-2문장 |
| **Q** (Question) | 자연스럽게 떠오르는 질문 | 1문장 (생략 가능) |
| **A** (Answer) | 핵심 결론/권고안 — **반드시 서두에** | 1-3문장 |

### 예시

> **S**: 우리 회사는 3년간 연 20% 성장을 기록해왔습니다.
> **C**: 그러나 주력 시장 세그먼트가 포화 징후를 보이고 있습니다.
> **Q**: 향후 5년간 성장을 유지하려면 어떻게 해야 할까요?
> **A**: 인접 헬스케어 시장에 진출하여 2028년까지 $2B 신규 매출을 목표로 해야 합니다.

## 본문: 피라미드 구조

```
Level 0: 핵심 메시지 (1개) ← SCQA의 A(Answer)
  │
  ├── Level 1: 핵심 근거 1 [요약 문장]
  │     ├── 데이터/증거 a
  │     ├── 데이터/증거 b
  │     └── 데이터/증거 c
  │
  ├── Level 1: 핵심 근거 2 [요약 문장]
  │     ├── 데이터/증거 a
  │     └── 데이터/증거 b
  │
  └── Level 1: 핵심 근거 3 [요약 문장]
        ├── 데이터/증거 a
        └── 데이터/증거 b
```

## 3가지 철칙

1. **모든 레벨의 아이디어는 하위 아이디어의 요약이어야 한다**
2. **같은 그룹의 아이디어는 논리적으로 같은 종류여야 한다**
3. **같은 그룹의 아이디어는 논리적 순서가 있어야 한다** (시간순 / 구조순 / 중요도순)

## 구체적 템플릿

```
# [보고서 제목]

## 요약
[SCQA: Situation → Complication → Answer. 3-5문장]

## 1. [핵심 근거 1의 요약 문장]
### 1.1 [증거/데이터]
### 1.2 [증거/데이터]

## 2. [핵심 근거 2의 요약 문장]
### 2.1 [증거/데이터]
### 2.2 [증거/데이터]

## 3. [핵심 근거 3의 요약 문장]
### 3.1 [증거/데이터]
### 3.2 [증거/데이터]

## 다음 단계
- [구체적 행동] — 담당: [이름] — 기한: [날짜]
```

## 품질 체크

- [ ] 결론이 첫 문단에 있는가?
- [ ] 핵심 근거가 3-5개인가?
- [ ] 모든 수치가 구체적인가? ("증가" → "12% 증가, $4.2M")
- [ ] 각 섹션 제목이 주장/결론 형태인가? (토픽 X)

## 프로액티브 트리거

- 결론이 문서 뒤쪽에 위치해 있으면 → 서두(Answer)로 이동 제안
- 근거 없이 주장만 나열되어 있으면 → 데이터/증거 보강 요청
- SCQA 4요소 중 빠진 요소가 있으면 → 누락된 요소 식별 및 추가 유도
- 논리 비약이 발견되면 → Level 간 연결 고리 보완 제안
- 같은 그룹의 아이디어가 논리적으로 다른 종류이면 → MECE 원칙 위반 지적
- 섹션 제목이 토픽형("시장 분석")이면 → 주장형("시장은 연 15% 성장 중")으로 변환 권고

## 산출물

- 다음 단계(Action Items) 포함 실행 계획표
- 품질 체크리스트 완료 보고
