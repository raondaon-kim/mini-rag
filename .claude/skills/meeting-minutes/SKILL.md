# 회의록 작성

## 표준 회의록 구조

```
# 회의록: [회의 제목]

- 일시: YYYY-MM-DD HH:MM ~ HH:MM
- 장소: [장소/화상 플랫폼]
- 참석자: [이름(역할), ...]
- 불참: [이름]
- 작성자: [이름]

---

## 안건 1: [주제]
- 논의 내용: [핵심 논점 요약]
- 결정 사항: [구체적으로 무엇이 결정되었는지]
- 액션 아이템:
  - [ ] [작업] — 담당: [이름] — 기한: [날짜]
  - [ ] [작업] — 담당: [이름] — 기한: [날짜]

## 안건 2: [주제]
- (동일 구조)

---

## 보류 사항 (다음 회의로 이월)
- [항목] — 논의 예정일: [날짜]

## 다음 회의
- 일시: [날짜]
- 안건: [이월 항목]
```

## DACI 의사결정 로그

중요한 결정은 별도 기록:

```
## 의사결정: [결정 제목]
- D (Driver): [추진 담당자]
- A (Approver): [최종 결정권자]
- C (Contributors): [의견 제공자들]
- I (Informed): [결과 통보 대상]

### 배경: [왜 이 결정이 필요한지]
### 검토 옵션:
| 옵션 | 장점 | 단점 | 예상 영향 |
|------|------|------|----------|
| A | ... | ... | ... |
| B | ... | ... | ... |

### 결정: [선택된 옵션 + 근거]
### 후속 조치: [담당자 + 기한]
```

## 회의록 작성 규칙
1. **결정 사항을 기록** — 논의 과정이 아님
2. **모든 액션아이템에 담당자+기한** 필수
3. **결정의 정확한 문구 사용** — 의역 금지
4. **24시간 이내 공유**

## 프로액티브 트리거
- 참석자 정보가 누락되어 있음
- 논의 내용만 있고 결정 사항이 불명확함
- 액션아이템에 담당자 또는 기한이 없음
- 다음 회의 일정이 기재되지 않음
- 안건과 결론이 일치하지 않음
