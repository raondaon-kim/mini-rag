# 번역/다국어 가이드

## 번역 규칙

### 일반 원칙
1. **의미 번역** — 단어 대 단어가 아닌 의미 단위 번역
2. **대상 언어의 자연스러운 표현** 사용
3. **전문 용어는 일관되게** — 용어집(Glossary) 유지
4. **문화적 맥락** 고려 — 직역이 어색한 관용구/숙어 처리

### 한→영 번역 시
- 존칭 처리: "~님" → 이름 그대로 또는 "Mr./Ms."
- 호칭: "부장님" → "Director [Name]" 또는 직접 이름
- 날짜: YYYY-MM-DD → Month DD, YYYY
- 숫자: 만/억 → thousand/million/billion

### 영→한 번역 시
- 기술 용어: 원어 병기 "인공지능(AI)"
- 외래어: 국립국어원 표기 기준
- 긴 문장: 한국어 기준으로 적절히 분리

## 용어집 관리

```
| 원문 (English) | 번역 (한국어) | 비고 |
|---------------|-------------|------|
| Revenue | 매출 | 순매출 = Net Revenue |
| Operating Income | 영업이익 | |
| Compliance | 컴플라이언스 | 외래어 그대로 |
| Stakeholder | 이해관계자 | |
```

**규칙**: 같은 문서 내에서 같은 원문은 항상 같은 번역 사용

## 번역 품질 체크
- [ ] 전문 용어가 용어집과 일치하는가?
- [ ] 원문의 의미가 정확히 전달되었는가?
- [ ] 대상 언어로 자연스러운가?
- [ ] 숫자, 날짜, 단위 형식이 대상 지역에 맞는가?
- [ ] 고유명사가 올바르게 처리되었는가?

## 프로액티브 트리거
- 같은 문서 내 같은 원문에 다른 번역 사용 → 용어 일관성 확인
- 직역으로 어색한 표현 → 의미 번역으로 전환
- 전문 용어 원어 병기 없음 → 초출 시 "한국어(English)" 병기
- 날짜/숫자 형식이 대상 지역에 안 맞음 → 현지화 형식 적용
- 문화적으로 부적절한 관용구/숙어 직역 → 대상 문화에 맞는 표현으로
